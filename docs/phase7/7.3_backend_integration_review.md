## 2. Integration Review Notes

### 2.1 Networking & Latencies
*Review finding:* The initial backend test suite hit timeouts because the backend `HttpClient` was set to a strict `50ms`. 
*Resolution:* While the ML model takes ~12ms, network latency added ~25ms, and rule-based queries added up to 80ms under high load. We increased the HTTP client timeout on the main Backend server to `500ms` as agreed in the SLA. Passes consistently now.

### 2.2 Date Formatting Exceptions
*Review finding:* Backend occasionally passed `DD-MM-YYYY` formats depending on the end-user's browser locale. This caused ML extraction failures for seasonal features.
*Resolution:* Backend was updated to strictly coerce ISO 8601 `YYYY-MM-DD` strings before forwarding to the Costing Engine.

### 2.3 Floating Point Precision
*Review finding:* Backend serialized the `totalCost` floats, occasionally ending up with rounding discrepancies (e.g. `1230.50` vs `1230.4999999`) causing unit test assertions to fail.
*Resolution:* Added `round(result, 2)` inside `ProductionPredictor` so the API always returns exactly 2 decimals natively to the backend JSON.
