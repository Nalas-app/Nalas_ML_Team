## 1. Overview
The ML Costing Engine is deployed as a standalone FastAPI microservice. It provides a deterministic cost breakdown (ingredient, labor, overhead, profit) for any menu item, utilizing a machine-learning **GradientBoosting** model with an automatic fast-fail **Rule-Based fallback** for 100% uptime reliability.

### Base URL (Local Development)
`http://localhost:8001`

---

## 2. Running the Service

### Prerequisites
Ensure Python 3.10+ is installed.

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Start the service
uvicorn ml_service.main:app --host 0.0.0.0 --port 8001
```

> **Note:** The service will automatically load the active ML model (`model/active/`) and dataset (`data/structured/`) into memory upon startup. This initial load takes ~1-2 seconds.

## 3. Core API Endpoints

### 3.1 Predict Cost [POST `/ml/predict-cost`]
The primary endpoint to generate a cost quotation breakdown.

**Request Body (`application/json`)**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `menuItemId` | str | ✅ Yes | ID or exact name of the menu item (e.g., `"ITEM-001"`) |
| `quantity` | int | ✅ Yes | Number of servings (Must be > 0) |
| `eventDate` | str | Optional | Format `YYYY-MM-DD` |
| `guestCount` | int | Optional | Expected total guests |

**Example Request:**
```json
{
  "menuItemId": "ITEM-001",
  "quantity": 100,
  "eventDate": "2026-11-20"
}
```

**Example Response (200 OK):**
```json
{
  "ingredientCost": 15000.0,
  "laborCost": 2250.0,
  "overheadCost": 1500.0,
  "totalCost": 20625.0,
  "confidence": 0.85,
  "modelVersion": "v1.0.0",
  "method": "ml_model"
}
```

### 3.2 List Available Menu Items [GET `/ml/menu-items`]
Returns all menu items and their data availability for predictions.

**Example Response Summary:**
```json
{
  "total": 34,
  "predictionReady": 26,
  "items": [
    {
      "itemId": "ITEM-001",
      "name": "Chicken Biryani",
      "predictionReady": true
    }
  ]
}
```

### 3.3 Health & Metrics
- **GET `/ml/health`**: Verifies DB connections and model availability.
- **GET `/ml/metrics`**: Returns prediction counts, error rates, and average latencies (useful for Prometheus/Grafana integration).

## 4. Integration Requirements for the Backend Team

### 4.1 Error Handling & Fallbacks
The ML engine returns specific HTTP status codes. Your backend system must handle these gracefully.

| Code | ML Error Code | Scenario | Backend Action Required |
|------|--------------|----------|-------------------------|
| `400` | `ML_001` | Invalid quantity (e.g., negative) | Return 400 validation error to frontend. |
| `400` | `ML_002` | Menu item ID not found | Return 404 item not found to frontend. |
| `404` | `ML_003`/`ML_004` | Item exists, but recipe/data is incomplete | Cannot quote this item automatically. Display "Quote on Request" fallback to user. |
| `500` | `ML_500` | Internal server or ML failure | Retry once. If persistent, raise server alert and fall back to manual quotation. |

### 4.2 Handling the `confidence` Score
Every response contains a `confidence` float `[0.10 — 0.95]`.

1. **Confidence < 0.60**: The ML Engine recommends **Manual Review**. Your backend system should save this quotation as "Pending Approval" rather than automatically sending it to the customer.
2. **Confidence > 0.60**: Safe to auto-approve.

### 4.3 Log Tracking
The ML Engine writes local `.jsonl` logs for retraining purposes into `logs/predictions/`. However, your backend should also save the `modelVersion` and `method` into your *Orders* database table to track which model generated which quote over time.

## 5. Performance Optimization Tips
1. **Network Latency:** The ML Core responds in **~15ms API time**. Host this microservice in the same VPC/Subnet as your core backend to avoid network overhead.
2. **Batching:** If a customer orders 5 different menu items, send 5 parallel asynchronous `POST` requests. The Engine is multi-threaded and will handle them faster than sequential loops.
3. **Timeouts:** Set your backend HTTP client timeout to `500ms`. If it takes longer, cancel the request and trigger a fallback.
