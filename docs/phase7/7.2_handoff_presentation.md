## 1. Model Overview (15 min)

### The Problem
Generating instant, accurate quotes for 34 catering menu items based on exact ingredient combinations without manual chef intervention.

### Our Solution
A **Hybrid Architecture**:
1. **Primary Model:** GradientBoostingRegressor (Predicts actual nonlinear variation of ingredients vs. volume).
2. **Fallback Model:** Rule-Based Engine (Calculates deterministic sub-totals purely off database multipliers).
3. **Safety Layer:** The API intelligently routes between the two using a 4-signal `Confidence` logic.

### Performance
| Metric | Synthetic Baseline | ML Test Data |
|--------|----------------|---------------|
| **MAPE** (Error) | `14.49%` | Target `< 10%` after week 1 data collection. |
| **R²** (Variance) | `0.975` | Captures `>97%` of real cost changes. |
| **Latency** | `15ms` API avg. | Target `< 500ms`. |

### Limitations
1. Assumes current ingredient prices (CSV loaded on boot).
2. Does **not** handle custom unlisted items.
3. Will have wider errors for quantities > 2000 until we collect larger historic batches.

## 2. Code Walkthrough (30 min)

### 2.1 The Prediction Engine (`ml_service/production_predictor.py`)
- Discuss `ProductionPredictor` singleton. Is thread-safe.
- Logic: `try -> ML predict -> verify confidence > 0.40 -> return`
- If confidence drops or ML fails, it automatically reroutes through `cost_engine.py` (Rule Base).

### 2.2 Feature Engineering (`feature_pipeline.py`)
- Extracts raw variables (Menu ID, Quantity, Guest Date) -> Transmutes to `base_ingredient_cost`, `is_wedding_season`, `month_sin`, etc.
- Drops features through `MinMaxScaler` and cyclical encoders.

### 2.3 Error Handling (`main.py`)
- We have wrapped everything into clean Fast API error codes.
- E.g. `400 ML_002` = Menu Item ID missing.
- E.g. `404 ML_003` = The Database recipe is missing an ingredient link.
