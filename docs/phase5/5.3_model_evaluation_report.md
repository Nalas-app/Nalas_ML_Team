## 2. Model Comparison Table

| Model | MAPE ↓ | RMSE | R² ↑ | Train (ms) | Pred (ms) | CV R² |

| Linear Regression | 25,682% | 25,255,590 | 0.0290 | 4.1 | 0.011 | 0.6288 |

### Key Insights

1. **Baseline is strong** — Mean-cost-per-item with quantity scaling achieves 8.68% MAPE. This is because our synthetic data is generated FROM the rule-based engine + slight noise, so the mean captures the pattern well.

2. **Tree-based models perform well** — RF and GB both achieve MAPE under 15%, confirming the features are predictive.

3. **Linear models fail badly** — MAPE > 24,000% because the cost distribution is highly skewed (anomalous items like Mutton Varuval at Rs. 34L). Linear models cannot handle this non-linear relationship.

4. **Gradient Boosting (Tuned) is the best ML model** — 14.49% MAPE, R² = 0.9759, best balance of accuracy and generalization.

**Hyperparameters (after tuning):**
```
learning_rate: 0.05

```

### Selection Rationale
- **GB (Tuned)** offers the best MAPE among ML models (14.49%)
- Strong R² of 0.9759 — captures 97.6% of cost variance
- Training is fast (< 1 second) and prediction is instant (< 1ms)
- The baseline being slightly better (8.68%) is expected on synthetic data — with **real order data**, the ML model will learn patterns (seasonality, demand effects) that the simple mean cannot capture

## 6. Production Deployment Strategy

### Phase 1 (Now): Rule-Based MVP
- Use the deterministic cost engine (`cost_engine.py`)
- Log all predictions for training data (`prediction_logger.py`)
- Confidence capped at 85%

### Phase 2 (After 100+ real orders): Hybrid
- Use GB model for items with sufficient history (10+ orders)  
- Fall back to rule-based for items with < 10 orders
- Compare ML prediction vs rule-based, log both

### Phase 3 (After 500+ real orders): Full ML
- Retrain model with real data
- Expected MAPE improvement to < 10% with real patterns
- Phase out rule-based for covered items
