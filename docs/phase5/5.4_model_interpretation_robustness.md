## 2. Feature Relationship Analysis

### Cost vs Quantity
```
Expected: Roughly linear relationship (more servings = more cost)
Actual:   ✅ Confirmed — quantity is 2nd most important feature (24.1%)
Notes:    Bulk discounts create slight non-linearity at high quantities
```

### Cost vs Ingredient Cost
```
Expected: Strong positive correlation (higher ingredient cost = higher total)
Actual:   ✅ Confirmed — 63.4% importance, dominant predictor
Notes:    This validates that the recipe-based costing approach is sound
```

### Cost vs Season
```
Expected: Wedding season (Oct-Feb) slightly higher
Actual:   ⚠️ Weak signal (< 1% importance)
Notes:    On synthetic data seasonal effect is small. Will strengthen with real data.
```

### Cost vs Category
```
Expected: Non-veg > Veg
Actual:   ✅ Confirmed — category features combined ~3.7% importance
Notes:    Breakfast category is actually more important than expected
          (Pongal has anomalous high cost due to data issue)
```

## 4. Known Limitations & Recommendations

| Limitation | Impact | Mitigation |
|-----------|--------|-----------|
| Trained on synthetic data | Model learns rule-based patterns, not real ones | Retrain when 100+ real orders available |
| MAPE 14.49% (target < 10%) | Slightly above accuracy target | Will improve with real data and data cleaning |
| High-cost anomalies (Mutton, Pongal) | Skew RMSE significantly | Fix unit errors in source data |
| No historical demand feature | Cannot capture demand patterns | Start collecting order data immediately |
| No price history feature | Cannot capture price volatility | Start snapshotting ingredient prices monthly |
