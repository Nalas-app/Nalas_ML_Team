## 1. Dashboard Design & Layout

We recommend deploying a 4-panel architecture inside Datadog or Grafana pulling metrics directly from our API endpoints (`/ml/metrics` & `/ml/health`).

### Row 1: Model Accuracy & Performance (Primary Focus)
* **Metric 1: Rolling 7-Day MAPE (Gauge):** Target < 10%. Shows the core accuracy of our ML predicting model vs Actual Invoice database.
* **Metric 2: Rolling 30-Day MAPE (Line Chart):** Identifies slow seasonal drift in accuracy across weeks.
* **Metric 3: Error Distribution (Histogram):** X-axis is Absolute Error %. Identifies if we have a wide bell curve (bad) or a sharp spike near 0 (good).
* **Metric 4: Predictions vs Actuals (Scatter Plot):** X=Expected Cost, Y=Predicted Cost. A perfect 45-degree angle means 100% accuracy.

### Row 2: Operational Health
* **Predictions Volume (Bar trend):** Volume per day.
* **Prediction Latency (P95/P99 - Line):** Target < 500ms. Watch for DB timeout spikes.
* **Fallback Rate (Pie Chart):** Ratio of ML Model Predictions vs. Rule-Based. Target is >80% ML. If this drops below 50%, our ML accuracy confidence threshold (`0.40`) is failing too often.
* **API Availability:** Uptime percentage.

### Row 3: Data Quality Triggers
* **Ingredient Nulls:** Count of 0.0 values in the master dictionary matrix. Red flag if > 5.
* **Un-Predictable Items:** Menu items lacking base recipe matrices from the Chef's inputs.
* **Data Freshness:** Timestamp of the last processed actual invoice syncing to the feedback loop (`scripts/phase8_feedback_loop.py`).

## 2. Alert Escalation Playbook

All alerts trigger via PagerDuty (Webhook integration). 

### 🔴 Critical Alerts (Immediate Page-Out)
| Alert Rule | What it Means | Playbook Action |
|------------|---------------|-----------------|
| **MAPE > 20% for 3 Days** | The model is suddenly hallucinating extreme costs. | **ACTION:** Investigate `feedback_*.csv` logs to identify which Item is driving variance. If a new ingredient price spiked, run `scripts/phase6_retrain.py --force`. |
| **P99 Latency > 1.0s** | DB connections or CPU exhaustion. | **ACTION:** Verify memory limits on the Docker container. Restart container. |
| **API Down > 5 min** | Network failure or Process Crash. | **ACTION:** Wake up API Backend DevOps team. Check reverse proxy rules. |

### 🟠 Warning Alerts (Review Next Business Day)
| Alert Rule | What it Means | Playbook Action |
|------------|---------------|-----------------|
| **MAPE > 15% for 7 Days** | Slow data drift (seasonal or menu shifts). | **ACTION:** Monitor auto-retraining logs. If auto-retrain fails to improve it, schedule a manual hyperparameter review. |
| **Fallback Rate > 20%** | The ML model is dropping confidence frequently (Likely due to strange user quantities being asked). | **ACTION:** Query `error_analysis.csv`. Identify if we need to adjust the `0.40` threshold or train on larger batch sizes. |
| **Missing Prices Detected** | Procurement updated DB but left a NULL row. | **ACTION:** Alert Kitchen Ops. Provide list of `UNMATCHED` elements. |

### 🟢 Info Alerts
| Alert Rule | Action |
|------------|--------|
| **Auto-Retrain Success** | Record new model version `vX.X` in Slack channel `#ml-ops`. No action needed. |
| **Volume Spike (>2x Daily)** | Note if correlating to Marketing campaigns. Watch latency. |
