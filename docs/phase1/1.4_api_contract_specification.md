## 1. Overview

The ML Costing API is a RESTful service that provides cost predictions for catering menu items. It runs as a **separate FastAPI service** on port `8001` and communicates with the Nalas backend via HTTP/JSON.

**Base URL:** `http://localhost:8001`

---

## 2. Endpoints

### 2.1 Cost Prediction

**`POST /ml/predict-cost`**

Predicts the complete cost breakdown for a given menu item and quantity.

#### Request

**Headers:**
```
Content-Type: application/json
```

**Body Schema:**
```json
{
  "menuItemId": "string (uuid, required)",
  "quantity": "number (positive integer, required)",
  "eventDate": "string (ISO 8601 date, optional)",
  "guestCount": "number (positive integer, optional)"
}
```

**Field Details:**

| Field | Type | Required | Constraints | Description |
|-------|------|----------|-------------|-------------|
| `menuItemId` | string (UUID) | ✅ Yes | Must be a valid UUID; must exist in database | Identifier for the menu item |
| `quantity` | integer | ✅ Yes | Must be > 0 | Number of servings requested |
| `eventDate` | string (ISO 8601) | ❌ No | Must be a future date (format: `YYYY-MM-DD`) | Date of the event (used for seasonal pricing) |
| `guestCount` | integer | ❌ No | Must be > 0 if provided | Expected number of guests |

**Example Request:**
```json
{
  "menuItemId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "quantity": 200,
  "eventDate": "2026-06-15",
  "guestCount": 150
}
```

**Minimal Request (required fields only):**
```json
{
  "menuItemId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "quantity": 200
}
```

---

#### Success Response

**Status Code:** `200 OK`

**Body Schema:**
```json
{
  "ingredientCost": "number (>= 0, 2 decimal places)",
  "laborCost": "number (>= 0, 2 decimal places)",
  "overheadCost": "number (>= 0, 2 decimal places)",
  "totalCost": "number (> 0, 2 decimal places)",
  "confidence": "number (0.0 - 1.0, 2 decimal places)",
  "modelVersion": "string (semantic version)",
  "method": "string (enum: 'rule_based' | 'ml_model')"
}
```

**Field Details:**

| Field | Type | Description |
|-------|------|-------------|
| `ingredientCost` | number | Total ingredient cost (₹) after scaling and seasonal adjustment |
| `laborCost` | number | Labor cost = 15% of ingredient cost (₹) |
| `overheadCost` | number | Overhead cost = 10% of ingredient cost (₹) |
| `totalCost` | number | Final cost including profit margin (₹) |
| `confidence` | number | Confidence score (0.0 = no confidence, 1.0 = fully confident) |
| `modelVersion` | string | Version of model used (e.g., `"v0.1.0"`) |
| `method` | string | `"rule_based"` or `"ml_model"` — indicates prediction method |

**Example Success Response:**
```json
{
  "ingredientCost": 15000.00,
  "laborCost": 2250.00,
  "overheadCost": 1500.00,
  "totalCost": 20625.00,
  "confidence": 0.85,
  "modelVersion": "v0.1.0",
  "method": "rule_based"
}
```

---

#### Error Response

**Status Codes:**
- `400 Bad Request` — Validation error in request
- `404 Not Found` — Menu item or recipe not found
- `500 Internal Server Error` — Prediction service internal failure
- `503 Service Unavailable` — Service temporarily unavailable

**Body Schema:**
```json
{
  "error": {
    "code": "string",
    "message": "string",
    "details": "object (optional)"
  }
}
```

**Example Error Responses:**

**ML_001 — Invalid Menu Item ID:**
```json
{
  "error": {
    "code": "ML_001",
    "message": "Invalid menu item ID format",
    "details": {
      "field": "menuItemId",
      "received": "not-a-uuid",
      "expected": "Valid UUID v4 format"
    }
  }
}
```

**ML_002 — Invalid Quantity:**
```json
{
  "error": {
    "code": "ML_002",
    "message": "Quantity must be a positive number",
    "details": {
      "field": "quantity",
      "received": -5,
      "expected": "Integer > 0"
    }
  }
}
```

**ML_003 — Recipe Not Found:**
```json
{
  "error": {
    "code": "ML_003",
    "message": "Recipe not found for the specified menu item",
    "details": {
      "menuItemId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "suggestion": "Ensure the menu item has an associated recipe in the database"
    }
  }
}
```

**ML_004 — Ingredient Price Unavailable:**
```json
{
  "error": {
    "code": "ML_004",
    "message": "One or more ingredient prices are unavailable",
    "details": {
      "missingIngredients": ["saffron", "truffle_oil"],
      "availableIngredients": 12,
      "totalIngredients": 14,
      "suggestion": "Update ingredient prices in the database"
    }
  }
}
```

**ML_005 — Prediction Failed (Fallback Used):**
```json
{
  "error": {
    "code": "ML_005",
    "message": "ML model prediction failed, rule-based fallback was used",
    "details": {
      "originalError": "Model inference timeout",
      "fallbackMethod": "rule_based",
      "fallbackResult": {
        "ingredientCost": 15000.00,
        "laborCost": 2250.00,
        "overheadCost": 1500.00,
        "totalCost": 20625.00,
        "confidence": 0.70
      }
    }
  }
}
```

> **Note:** ML_005 includes the fallback result in `details` so the client can still use the prediction.

## 3. Error Code Registry

| Code | HTTP Status | Condition | Recoverable | Client Action |
|------|-------------|-----------|-------------|---------------|
| `ML_001` | 400 | Invalid menu item ID (not a valid UUID) | ✅ | Fix the menu item ID format |
| `ML_002` | 400 | Quantity is zero or negative | ✅ | Provide a positive quantity |
| `ML_003` | 404 | No recipe found for menu item in database | ✅ | Add recipe data for this item |
| `ML_004` | 404 | Ingredient price(s) missing in database | ✅ | Update ingredient prices |
| `ML_005` | 200* | ML model failed but fallback succeeded | ✅ | Use fallback result from details |
| `ML_006` | 400 | Event date is in the past | ✅ | Provide a future event date |
| `ML_007` | 400 | Guest count is zero or negative | ✅ | Provide a positive guest count |
| `ML_500` | 500 | Internal server error (unexpected) | ❌ | Retry or contact support |
| `ML_503` | 503 | Service temporarily unavailable | ❌ | Retry after delay |

> *ML_005 returns HTTP 200 because the prediction was successful via fallback.

## 4. Validation Rules

### 4.1 Request Validation

```python
from pydantic import BaseModel, Field, validator
from uuid import UUID
from datetime import date
from typing import Optional

class CostPredictionRequest(BaseModel):
    menuItemId: UUID                          # Required, valid UUID
    quantity: int = Field(..., gt=0)          # Required, must be > 0
    eventDate: Optional[date] = None          # Optional, ISO 8601 date
    guestCount: Optional[int] = Field(None, gt=0)  # Optional, must be > 0 if provided
    
    @validator('eventDate')
    def event_date_must_be_future(cls, v):
        if v is not None and v <= date.today():
            raise ValueError('Event date must be in the future')
        return v
```

### 4.2 Response Validation

```python
class CostPredictionResponse(BaseModel):
    ingredientCost: float = Field(..., ge=0)    # >= 0
    laborCost: float = Field(..., ge=0)          # >= 0
    overheadCost: float = Field(..., ge=0)       # >= 0
    totalCost: float = Field(..., gt=0)          # > 0
    confidence: float = Field(..., ge=0, le=1)   # 0.0 - 1.0
    modelVersion: str                            # e.g., "v0.1.0"
    method: str                                  # "rule_based" or "ml_model"
```

### 4.3 Business Validation (Post-Prediction)

| Rule | Check | Action |
|------|-------|--------|
| All costs positive | `ingredientCost > 0 AND laborCost > 0 AND overheadCost > 0` | If any ≤ 0, raise ML_500 |
| Minimum threshold | `totalCost / quantity >= category_minimum` | If below, clamp to minimum |
| Variance check | `abs(totalCost - historical_avg) / historical_avg < 0.30` | If exceeds, add warning flag |
| Confidence check | `confidence >= 0.60` | If below, add "low_confidence" flag |
