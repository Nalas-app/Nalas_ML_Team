## 2. Multiplier Percentages

| Component | Percentage | Base | Rationale |
|-----------|-----------|------|-----------|
| **Labor Cost** | 15% | Ingredient Cost | Covers chef wages, helper wages, preparation time |
| **Overhead Cost** | 10% | Ingredient Cost | Covers utilities, equipment depreciation, rent allocation |
| **Profit Margin** | 10% | Subtotal (Ingredient + Labor + Overhead) | Standard catering industry profit margin |

### 2.1 Multiplier Adjustment Authority
- Multiplier percentages are **fixed defaults** set by management.
- Changes to multipliers require **written approval** from the business owner.
- Any multiplier change triggers a **model retraining** or **rule reconfiguration**.
- Multiplier changes are logged with timestamp and approver.

### 2.2 Calculation Formula (Pseudocode)
```python
def calculate_total_cost(ingredient_cost, season=None):
    # Step 1: Apply seasonal adjustment to ingredient cost
    adjusted_ingredient_cost = apply_seasonal_adjustment(ingredient_cost, season)
    
    # Step 2: Calculate component costs
    labor_cost = adjusted_ingredient_cost * 0.15
    overhead_cost = adjusted_ingredient_cost * 0.10
    
    # Step 3: Calculate subtotal
    subtotal = adjusted_ingredient_cost + labor_cost + overhead_cost
    
    # Step 4: Apply profit margin
    profit_margin = subtotal * 0.10
    
    # Step 5: Total cost
    total_cost = subtotal + profit_margin
    
    return {
        "ingredient_cost": adjusted_ingredient_cost,
        "labor_cost": labor_cost,
        "overhead_cost": overhead_cost,
        "subtotal": subtotal,
        "profit_margin": profit_margin,
        "total_cost": total_cost
    }
```

## 3. Seasonal Variations

### 3.1 Season Definitions

| Season | Calendar Months | Typical Events | Demand Level |
|--------|----------------|----------------|-------------|
| **Wedding Season** | October – February | Weddings, Receptions, Engagement ceremonies | **High** |
| **Festival Season** | March – April | Pongal, Tamil New Year, Navratri, Ugadi | **Medium-High** |
| **Off-Season** | May – September | Birthdays, Corporate events, small gatherings | **Normal (Baseline)** |

### 3.2 Seasonal Adjustment Factors

| Season | Ingredient Cost Adjustment | Rationale |
|--------|---------------------------|-----------|
| **Wedding Season** | +15% (range: +10% to +20%) | High demand for flowers, ghee, dry fruits; vendor price surges |
| **Festival Season** | +10% (range: +5% to +15%) | Moderate demand increase; specific ingredient spikes (jaggery, coconut) |
| **Off-Season** | 0% (baseline) | Normal market pricing |

### 3.3 Seasonal Adjustment Rules
1. Seasonal adjustment is applied to **ingredient cost only**, before labor/overhead multipliers.
2. The adjustment applies based on the **event date**, not the order date.
3. If no event date is provided, **no seasonal adjustment** is applied (assume baseline).
4. Overlapping seasons (e.g., February wedding + festival): use the **higher** adjustment.

## 4. Minimum Cost Thresholds

### 4.1 Per-Serving Minimums by Category

| Item Category | Minimum Cost Per Serving (₹) | Rationale |
|---------------|------------------------------|-----------|
| **Vegetarian Main Course** | 50 | Minimum viable ingredient + preparation cost |
| **Non-Vegetarian Main Course** | 100 | Higher base ingredient cost (chicken, mutton, fish) |
| **Rice / Biryani** | 60 | Rice + spice + preparation cost |
| **Dessert / Sweet** | 40 | Sugar, dairy, preparation cost |
| **Starter / Appetizer** | 45 | Moderate ingredient + frying/grilling cost |
| **Beverage** | 20 | Lowest preparation complexity |
| **Bread / Roti** | 25 | Flour + cooking fuel |
| **Side Dish / Raita** | 30 | Minimal ingredients |

### 4.2 Minimum Cost Enforcement Rules
1. If a prediction falls **below** the category minimum, the cost is **clamped** to the minimum.
2. A clamped prediction is **flagged** with a warning: `"cost_clamped_to_minimum"`.
3. Clamped predictions are logged separately for review (may indicate stale ingredient prices).
4. Minimum thresholds are reviewed **quarterly** and updated based on market conditions.

## 5. Maximum Acceptable Variance

### 5.1 Variance Detection Rules

| Scenario | Threshold | Action |
|----------|-----------|--------|
| Prediction vs historical average (same item, similar quantity) | > ±30% | Flag for manual review |
| Ingredient price change (single ingredient) | > ±50% | Alert + investigate price data |
| Ingredient price change (batch, > 10 items) | > ±20% | Trigger retraining review |
| Prediction vs previous prediction (same inputs) | > ±5% | Investigate model drift |

### 5.2 Historical Comparison Logic
```python
def check_variance(predicted_cost, menu_item_id, quantity):
    """
    Check if prediction is within acceptable variance of historical data.
    """
    # Get historical average for similar predictions
    historical_avg = get_historical_average(menu_item_id, quantity_range=(quantity * 0.8, quantity * 1.2))
    
    if historical_avg is None:
        # No historical data → cannot check variance
        return {"variance_check": "skipped", "reason": "no_historical_data"}
    
    variance_pct = abs(predicted_cost - historical_avg) / historical_avg * 100
    
    if variance_pct > 30:
        return {
            "variance_check": "flagged",
            "variance_pct": variance_pct,
            "historical_avg": historical_avg,
            "action": "manual_review_required"
        }
    
    return {"variance_check": "passed", "variance_pct": variance_pct}
```

## 6. Special Business Rules

### 6.1 Quantity-Based Discounts (Bulk Orders)

| Quantity (Servings) | Discount on Ingredient Cost |
|--------------------|-----------------------------|
| 1 – 100 | 0% (standard pricing) |
| 101 – 300 | 5% discount |
| 301 – 500 | 8% discount |
| 501 – 1000 | 10% discount |
| 1000+ | 12% discount (max) |

> Bulk discounts reflect wholesale purchasing power. Applied **before** labor/overhead multipliers.

### 6.2 New Menu Item Handling
- If a menu item has **no recipe data**, the system returns error code `ML_003`.
- If a menu item has a recipe but **missing ingredient prices**, error code `ML_004` is returned.
- New items must have complete recipe + pricing data before predictions can be generated.

### 6.3 Price Staleness Rules
- Ingredient prices are considered **stale** if not updated within **30 days**.
- Stale prices trigger a **warning** in the prediction response.
- Predictions with > 3 stale ingredients are flagged for manual review.

### 6.4 Rounding Rules
- All currency values are rounded to **2 decimal places**.
- Ingredient quantities are rounded to **3 decimal places** (grams/ml precision).
- Confidence scores are rounded to **2 decimal places**.

## 7. Error Handling Business Rules

| Error Code | Condition | User Message | Fallback |
|-----------|-----------|-------------|----------|
| `ML_001` | Invalid menu item ID (not a valid UUID) | "Invalid menu item identifier" | None, reject request |
| `ML_002` | Quantity ≤ 0 | "Quantity must be a positive number" | None, reject request |
| `ML_003` | No recipe found for given menu item ID | "Recipe data not available for this item" | None, reject request |
| `ML_004` | One or more ingredient prices unavailable | "Some ingredient prices are missing" | Use last known price if available |
| `ML_005` | Model prediction failed | "Prediction service temporarily unavailable" | Fall back to rule-based calculation |
