## 2. Layer-by-Layer Architecture

### 2.1 Data Layer

**Purpose:** Stores and serves all data required for cost prediction.

```
┌─────────────────────────────────────────────────────────┐
│                      DATA LAYER                         │
├─────────────────────────────────────────────────────────┤
│                                                         │

│  │  Recipe Data   │  │ Ingredient    │  │ Historical  │ │
│  │               │  │ Prices        │  │ Orders      │ │
│  │ - menu_item_id│  │               │  │             │ │
│  │ - ingredients │  │ - ingredient_ │  │ - order_id  │ │
│  │ - quantities  │  │   id          │  │ - menu_item │ │
│  │ - base_serving│  │ - name        │  │ - quantity  │ │
│  │ - category    │  │ - unit        │  │ - predicted │ │
│  │ - complexity  │  │ - price_per_  │  │ - actual    │ │
│  │               │  │   unit        │  │ - event_date│ │
│  │               │  │ - last_update │  │ - created_at│ │

│  │               │  │               │                   │
```

**Data Sources:**
| Source | Type | Update Frequency |
|--------|------|-----------------|
| Recipe Master | Static (manual updates) | On menu change |
| Ingredient Prices | Semi-static | Weekly to monthly |
| Historical Orders | Transactional | Per completed order |
| Prediction Logs | Append-only | Per prediction request |
| Model Registry | Reference | Per model deployment |

### 2.2 Feature Engineering Layer

**Purpose:** Transforms raw data into model-ready features.

```
┌─────────────────────────────────────────────────────────────────┐
│                   FEATURE ENGINEERING LAYER                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  RAW INPUTS                  ENGINEERED FEATURES                │
│  ──────────                  ──────────────────                 │
│                                                                 │
│  menu_item_id ──────────▶  ingredient_count                    │
│  recipe data  ──────────▶  total_ingredient_cost               │
│                          ▶  recipe_complexity_score             │
│                          ▶  avg_ingredient_price                │
│                          ▶  max_ingredient_price                │
│                          ▶  category_encoded (one-hot)          │
│                                                                 │
│  quantity     ──────────▶  quantity (raw)                       │
│                          ▶  quantity_bucket (small/med/large)   │
│                          ▶  bulk_discount_factor                │
│                                                                 │
│  event_date   ──────────▶  month                               │
│                          ▶  day_of_week                         │
│                          ▶  season (wedding/festival/off)       │
│                          ▶  is_weekend                          │
│                          ▶  seasonal_adjustment_factor          │
│                          ▶  days_until_event                    │
│                                                                 │
│  guest_count  ──────────▶  guest_count (raw)                   │
│                          ▶  servings_per_guest                  │
│                          ▶  event_scale (small/med/large)       │
│                                                                 │
│  DERIVED      ──────────▶  price_volatility_30d                │
│  (from historical data)  ▶  demand_index (from order frequency)│
│                          ▶  avg_cost_historical (same item)    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**Feature Categories:**

| Category | Features | Available In |
|----------|----------|-------------|
| **Recipe Features** | ingredient_count, total_ingredient_cost, recipe_complexity_score, avg/max ingredient price | Phase 1 (Rule-Based) |
| **Quantity Features** | quantity, quantity_bucket, bulk_discount_factor | Phase 1 |
| **Time Features** | month, day_of_week, season, is_weekend, seasonal_adjustment_factor | Phase 1 |
| **Demand Features** | demand_index, price_volatility_30d | Phase 3+ (requires data) |
| **Historical Features** | avg_cost_historical, prediction_accuracy_trend | Phase 3+ (requires data) |

## 3. Technology Stack

| Layer | Technology | Reason |
|-------|-----------|--------|
| **ML Service** | Python 3.10+ / FastAPI | Industry standard for ML services; async support |
| **ML Libraries** | scikit-learn, XGBoost, pandas, numpy | Mature, well-documented ML ecosystem |
| **Model Serialization** | joblib / pickle | Standard for scikit-learn models |
| **API Server** | Uvicorn (ASGI) | High-performance async server |
| **Data Validation** | Pydantic v2 | Type-safe request/response handling |
| **Backend** | Node.js (existing) | Existing Nalas backend application |
| **Database** | PostgreSQL / MongoDB (existing) | Existing data store |
| **Logging** | Python logging (structured JSON) | Built-in, extensible |
| **Monitoring** | Custom metrics endpoint | Lightweight, no extra infra needed |

## 4. Communication Architecture

```
┌─────────────┐         HTTP/REST          ┌──────────────┐
│             │  POST /ml/predict-cost      │              │
│   Backend   │ ──────────────────────────▶ │  ML Service  │
│  (Node.js)  │                             │  (FastAPI)   │
│             │ ◀────────────────────────── │              │
│             │     JSON Response           │              │
└─────────────┘                             └──────────────┘
     Port: 3000                                Port: 8001
```

**Communication Protocol:**
- **Synchronous REST API** calls from backend to ML service
- **JSON** request/response format
- **Timeout:** Backend sets 5-second timeout for ML service calls
- **Retry:** 1 retry with 500ms delay on timeout/5xx errors
- **Circuit Breaker:** After 5 consecutive failures, backend uses rule-based fallback for 60 seconds
