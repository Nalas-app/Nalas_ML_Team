## 1. Monitoring Metrics Specification

### 1.1 Model Performance Metrics

| Metric | Definition | Calculation | Frequency |
|--------|-----------|-------------|-----------|
| **Prediction Accuracy (MAPE)** | Mean Absolute Percentage Error between predicted and actual costs | `mean(\|actual - predicted\| / actual) × 100` | Rolling 30-day window |
| **MAE** | Average absolute prediction error in ₹ | `mean(\|actual - predicted\|)` | Rolling 30-day window |
| **RMSE** | Root mean squared error in ₹ | `sqrt(mean((actual - predicted)²))` | Rolling 30-day window |
| **R² Score** | Coefficient of determination | `1 - SS_res / SS_tot` | Per model version |
| **Prediction Error Distribution** | Distribution of (predicted - actual) errors | Histogram + percentiles | Weekly |
| **Confidence Score Distribution** | Distribution of confidence scores across predictions | Histogram + mean/median | Daily |
| **Model Drift** | Shift in feature distributions vs training data | KL Divergence / PSI score | Weekly |

### 1.2 Operational Metrics

| Metric | Definition | Calculation | Frequency |
|--------|-----------|-------------|-----------|
| **Predictions Per Day** | Total prediction requests received | Count of API calls | Daily |
| **Average Prediction Latency** | Mean response time | `mean(response_time_ms)` | Rolling 24h |
| **P95 Latency** | 95th percentile response time | `percentile(response_time_ms, 95)` | Rolling 24h |
| **P99 Latency** | 99th percentile response time | `percentile(response_time_ms, 99)` | Rolling 24h |
| **API Error Rate** | Percentage of failed requests (4xx + 5xx) | `error_count / total_count × 100` | Rolling 24h |
| **Fallback Usage Rate** | % of predictions using rule-based fallback when ML model is primary | `fallback_count / total_count × 100` | Rolling 24h |
| **Service Uptime** | Percentage of time service is available | `uptime_minutes / total_minutes × 100` | Monthly |

### 1.3 Data Quality Metrics

| Metric | Definition | Detection Method | Frequency |
|--------|-----------|------------------|-----------|
| **Missing Ingredient Prices** | Ingredients in recipes without price data | Query: recipes with unpriced ingredients | Daily |
| **New Menu Items Without Recipes** | Menu items added without recipe data | Query: menu items with no linked recipe | Daily |
| **Ingredient Price Volatility** | Price changes exceeding 50% in 30 days | Compare current vs 30-day-ago prices | Weekly |
| **Recipe Data Completeness** | % of recipes with all required fields | `complete_recipes / total_recipes × 100` | Weekly |
| **Stale Prices** | Prices not updated in > 30 days | `current_date - last_updated > 30 days` | Daily |
| **Order Data Completeness** | % of completed orders with actual cost recorded | `orders_with_actual / completed_orders × 100` | Daily |

## 2. Alert Rules Configuration

### 2.1 Critical Alerts (Immediate Action Required)

| Alert ID | Condition | Duration | Severity | Action |
|----------|-----------|----------|----------|--------|
| `CRIT-001` | MAPE > 20% | 3 consecutive days | 🔴 Critical | Investigate model performance; consider rollback |
| `CRIT-002` | API latency P99 > 1 second | 1 hour sustained | 🔴 Critical | Check server resources; restart service |
| `CRIT-003` | API error rate > 5% | 30 minutes sustained | 🔴 Critical | Check logs; investigate error patterns |
| `CRIT-004` | ML service down (health check failing) | 5 minutes | 🔴 Critical | Restart service; notify backend team |
| `CRIT-005` | Prediction returns negative cost | Single occurrence | 🔴 Critical | Investigate; this should never happen |

### 2.2 Warning Alerts (Investigation Required)

| Alert ID | Condition | Duration | Severity | Action |
|----------|-----------|----------|----------|--------|
| `WARN-001` | MAPE > 15% | 7 consecutive days | 🟡 Warning | Review model performance; plan retraining |
| `WARN-002` | Fallback usage rate > 20% | 24 hours | 🟡 Warning | ML model may be degrading; investigate |
| `WARN-003` | Missing ingredient price detected | On detection | 🟡 Warning | Notify data team to update prices |
| `WARN-004` | API latency P99 > 500ms | 1 hour sustained | 🟡 Warning | Monitor; may need resource scaling |
| `WARN-005` | Ingredient price change > 50% | On detection | 🟡 Warning | Verify price is correct; not data entry error |
| `WARN-006` | Prediction confidence < 60% rate > 30% | 24 hours | 🟡 Warning | Model may need retraining or more data |
| `WARN-007` | New menu item without recipe added | On detection | 🟡 Warning | Recipe data needed before predictions possible |
| `WARN-008` | Stale prices (> 30 days old) > 10% of ingredients | Weekly check | 🟡 Warning | Schedule price update with kitchen team |

### 2.3 Informational Alerts

| Alert ID | Condition | Severity | Action |
|----------|-----------|----------|--------|
| `INFO-001` | Model retraining completed | ℹ️ Info | Review metrics; approve for deployment |
| `INFO-002` | New model version deployed | ℹ️ Info | Monitor for 48 hours post-deployment |
| `INFO-003` | Prediction volume milestone (100, 500, 1000) | ℹ️ Info | Log for project tracking |
| `INFO-004` | Data collection milestone reached | ℹ️ Info | Evaluate readiness for next ML phase |

## 4. Incident Response Procedures

### 4.1 Severity Definitions

| Severity | Definition | Response Time | Examples |
|----------|-----------|---------------|---------|
| **P0 — Critical** | Service completely down or producing critically wrong results | < 15 minutes | Service crash, negative costs |
| **P1 — High** | Service degraded but partially functional | < 1 hour | High error rate, excessive latency |
| **P2 — Medium** | Non-critical issue affecting quality | < 4 hours | Rising MAPE, stale prices |
| **P3 — Low** | Minor issue, no immediate impact | Next business day | Missing recipe for rarely ordered item |

| Issue | First Action | Second Action | Escalation |
|-------|-------------|---------------|------------|
| **Service down** | Restart: `uvicorn main:app --port 8001` | Check logs for crash reason | Backend team (they see 503s) |
| **High error rate** | Check `/ml/metrics` for error breakdown | Fix data issues or rollback | Team lead |
| **High MAPE** | Check if ingredient prices changed | Check prediction logs for patterns | Schedule retraining review |
| **High latency** | Check server CPU/memory | Reduce worker count or restart | Infrastructure team |
| **Data quality** | Identify missing data items | Notify operations team | Team lead |

## 5. Metrics Collection Implementation

### 5.1 Prediction Logger (Python)

```python
import json
import time
import logging
from datetime import datetime
from pathlib import Path

class PredictionLogger:
    """Logs every prediction for monitoring and future model training."""
    
    def __init__(self, log_dir: str = "./logs/predictions"):
        self.log_dir = Path(log_dir)
        self.log_dir.mkdir(parents=True, exist_ok=True)
        self.logger = logging.getLogger("prediction_logger")
    
    def log_prediction(self, request, response, method, latency_ms, flags=None):
        log_entry = {
            "prediction_id": str(uuid.uuid4()),
            "timestamp": datetime.utcnow().isoformat(),
            "request": {
                "menuItemId": str(request.menuItemId),
                "quantity": request.quantity,
                "eventDate": str(request.eventDate) if request.eventDate else None,
                "guestCount": request.guestCount
            },
            "response": {
                "ingredientCost": response.ingredientCost,
                "laborCost": response.laborCost,
                "overheadCost": response.overheadCost,
                "totalCost": response.totalCost,
                "confidence": response.confidence
            },
            "method": method,
            "modelVersion": response.modelVersion,
            "latencyMs": latency_ms,
            "flags": flags or []
        }
        
        # Write to daily log file
        date_str = datetime.utcnow().strftime("%Y-%m-%d")
        log_file = self.log_dir / f"predictions_{date_str}.jsonl"
        
        with open(log_file, "a") as f:
            f.write(json.dumps(log_entry) + "\n")
        
        # Also log to structured logger for real-time monitoring
        self.logger.info("prediction", extra=log_entry)
```

### 5.2 Metrics Aggregator

```python
class MetricsAggregator:
    """Aggregates prediction metrics for the /ml/metrics endpoint."""
    
    def __init__(self):
        self.predictions_total = 0
        self.predictions_today = 0
        self.latencies = []        # Rolling window
        self.errors_by_code = {}
        self.method_counts = {"rule_based": 0, "ml_model": 0}
        self.fallback_count = 0
    
    def record_prediction(self, method, latency_ms, error_code=None):
        self.predictions_total += 1
        self.predictions_today += 1
        self.latencies.append(latency_ms)
        self.method_counts[method] += 1
        
        if error_code:
            self.errors_by_code[error_code] = self.errors_by_code.get(error_code, 0) + 1
        
        if method == "rule_based" and self.method_counts.get("ml_model", 0) > 0:
            self.fallback_count += 1
    
    def get_metrics(self):
        import numpy as np
        latencies = np.array(self.latencies[-1000:])  # Last 1000
        
        return {
            "predictions": {
                "total": self.predictions_total,
                "today": self.predictions_today,
                "method_breakdown": self.method_counts
            },
            "latency": {
                "p50_ms": float(np.percentile(latencies, 50)) if len(latencies) > 0 else 0,
                "p95_ms": float(np.percentile(latencies, 95)) if len(latencies) > 0 else 0,
                "p99_ms": float(np.percentile(latencies, 99)) if len(latencies) > 0 else 0
            },
            "errors": {
                "total": sum(self.errors_by_code.values()),
                "rate_pct": sum(self.errors_by_code.values()) / max(self.predictions_total, 1) * 100,
                "by_code": self.errors_by_code
            },
            "fallback": {
                "total": self.fallback_count,
                "rate_pct": self.fallback_count / max(self.predictions_total, 1) * 100
            }
        }
```

## 6. Drift Detection Strategy

### 6.1 Feature Drift Detection

| Feature | Detection Method | Threshold | Check Frequency |
|---------|-----------------|-----------|----------------|
| `ingredient_cost` distribution | Population Stability Index (PSI) | PSI > 0.2 → significant drift | Weekly |
| `quantity` distribution | PSI | PSI > 0.2 | Weekly |
| `seasonal_factor` distribution | PSI | PSI > 0.1 | Monthly |
| Overall prediction distribution | KL Divergence | KLD > 0.1 | Weekly |

### 6.2 Concept Drift Detection

```
If rolling 30-day MAPE increases by > 5 percentage points from 
the model's initial test MAPE → flag for retraining review.

Example:
  - Model trained with MAPE = 8%
  - Rolling MAPE reaches 13% (increase of 5pp)
  - → Trigger WARN-001 alert
  - → Schedule retraining review
```
