## 1. System Overview

### Purpose
The Rule-Based Costing Engine provides immediate, deterministic cost predictions for Nalas catering menu items. It serves as the Phase 1 MVP while collecting real order data for future ML model training.

### Architecture
```
Client (Backend)  →  POST /ml/predict-cost  →  FastAPI  →  CostEngine  →  Response
                                                   ↕              ↕
                                            PredictionLogger   DataStore
                                             (JSONL logs)     (CSV data)
                                                              BusinessRules
                                                             (JSON config)
```

### Technology Stack
| Component | Technology |
|-----------|-----------|
| Framework | FastAPI 0.115+ |
| Server | Uvicorn |
| Data | Pandas (CSV files) |
| Validation | Pydantic v2 |
| Language | Python 3.10+ |
| Port | 8001 |

## 2. API Documentation

### Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/ml/predict-cost` | POST | Cost prediction for a menu item |
| `/ml/health` | GET | Service health check |
| `/ml/model-info` | GET | Active model information |
| `/ml/metrics` | GET | Prediction performance metrics |
| `/ml/menu-items` | GET | List all menu items with readiness |
| `/` | GET | Root greeting |
| `/docs` | GET | Swagger UI (auto-generated) |

### POST /ml/predict-cost

**Request:**
```json
{
    "menuItemId": "ITEM-001",
    "quantity": 25,
    "eventDate": "2026-11-20",
    "guestCount": 100
}
```

**Successful Response (200):**
```json
{
    "ingredientCost": 63026.25,
    "laborCost": 9453.94,
    "overheadCost": 6302.63,
    "totalCost": 103646.67,
    "confidence": 0.62,
    "modelVersion": "v0.1.0",
    "method": "rule_based"
}
```

**Error Response (400/404/500):**
```json
{
    "error": {
        "code": "ML_002",
        "message": "Menu item not found: FAKE-ITEM",
        "details": {"menu_item_id": "FAKE-ITEM"}
    }
}
```

### Error Codes

| Code | HTTP Status | Meaning |
|------|------------|---------|
| ML_001 | 400 | Invalid input (quantity <= 0) |
| ML_002 | 400 | Menu item not found |
| ML_003 | 404 | Recipe not found for item |
| ML_004 | 404 | Cannot calculate cost (no priced ingredients) |
| ML_005 | 500 | ML model error (future use) |
| ML_500 | 500 | Internal server error |

## 3. Operational Guide

### 3.1 Deployment

**Start the server:**
```bash
cd nalas_app/ML_costing/ml_service
python -m uvicorn main:app --host 0.0.0.0 --port 8001
```

**Start with auto-reload (development):**
```bash
python -m uvicorn main:app --host 0.0.0.0 --port 8001 --reload
```

**Verify it's running:**
```bash
# Health check
GET http://localhost:8001/ml/health
```

### 3.2 Updating Ingredient Prices

1. Open `data/structured/ingredients_master.csv`
2. Find the ingredient row by name or ID
3. Update the `price_per_unit` column
4. Update `last_updated` date
5. Restart the server (data is loaded at startup)

**Example:** Update Chicken Boneless price
```csv
ING-019,Chicken - BONELESS,Meat & Poultry,KG,350.0,False,2026-04-01
```

### 3.3 Adding a New Menu Item

1. Add row to `data/structured/menu_items.csv`:
```csv
ITEM-035,New Dish Name,Category,1 batch,Manual entry,Manual,True,False
```

2. Add recipe rows to `data/structured/recipes.csv`:
```csv
RCP-0587,ITEM-035,ingredient_name,ING-xxx,0.5,1.05
RCP-0588,ITEM-035,another_ingredient,ING-yyy,1.0,1.05
```

3. Restart the server

### 3.4 Changing Business Rules

Edit `ml_service/config/business_rules.json`:
```json
{
    "labor_pct": 0.15,       // Change labor percentage
    "overhead_pct": 0.10,    // Change overhead percentage
    "profit_pct": 0.10       // Change profit margin
}
```
Restart the server after changes.

### 3.5 Monitoring

**Check metrics:**
```
GET http://localhost:8001/ml/metrics
```

**Check prediction logs:**
```
# Daily logs stored in:
nalas_app/ML_costing/logs/predictions/predictions_YYYY-MM-DD.jsonl
```

**Check application logs:**
```
nalas_app/ML_costing/logs/app.log
```

## 4. Troubleshooting Guide

| Problem | Cause | Fix |
|---------|-------|-----|
| Server won't start | Missing dependencies | `pip install -r requirements.txt` |
| Server won't start | Data files missing | Run `python scripts/phase2_data_extraction.py` |
| ML_002 for valid item | Item not in CSV | Add to `menu_items.csv`, restart |
| ML_003 for valid item | Recipe not in CSV | Add recipe rows to `recipes.csv`, restart |
| ML_004 for valid item | No priced ingredients | Fix ingredient matching in `recipes.csv` |
| High latency | Large CSV files | Consider SQLite for > 1000 items |
| All costs identical | Low ingredient matching | Improve fuzzy matching in extraction script |
| Prediction log empty | No predictions made yet | Make a test prediction |
| Cost seems too high | Unit mismatch | Check recipe quantities (grams vs KG) |
| Cost seems too low | Missing ingredients | Check ingredient match rate in confidence |

## 5. Known Limitations

1. **Static data** — Prices and recipes are loaded from CSV at startup. Changes require server restart.
2. **No authentication** — API is open. Add JWT/API key before production deployment.
3. **No database** — Uses CSV files. Migrate to PostgreSQL for production.
4. **Partial recipe data** — Only 20.6% of menu items have complete quantity data.
5. **No real-time price updates** — Prices must be manually updated in CSV.
6. **Single-threaded on Windows** — Uvicorn on Windows may use single worker.
